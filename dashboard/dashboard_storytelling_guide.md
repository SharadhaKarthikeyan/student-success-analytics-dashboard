# Dashboard Storytelling & Interview Guide

This guide helps you explain the "Why" behind the dashboard to recruiters and hiring managers.

## 1. Project Purpose
The **Student Success Dashboard** was built to bridge the gap between administrative data and student interventions. While many universities track GPA, this dashboard integrates **operational bottlenecks** (like financial holds and testing delays) to predict and prevent student dropouts before they happen.

## 2. Target Audience
- **Dean of Students:** To monitor overall graduation and dropout health.
- **Academic Advisors:** To identify which students to call for "At-Risk Outreach" today.
- **Registrar Office:** To identify friction points in the enrollment process (e.g., slow hold resolution).

## 3. Key Decision Support
- **Decision:** Where should we allocate the budget for new advisors? 
  - *Data Support:* Page 2 shows which majors have the highest risk scores.
- **Decision:** Should we change our math placement test vendor?
  - *Data Support:* Page 3 shows if specific languages have unusually high retake rates.

## 5. LinkedIn Project Description
**Student Success Analytics Dashboard: Identifying At-Risk Students**
Building a predictive retention model is only half the battle; the other half is making it actionable. I built this dashboard to help university administrators visualize the "invisible" hurdles students face, from financial holds to placement test retakes. 

Using a combination of Python for risk-scoring logic and Power BI/Tableau for visualization, I created a 4-page analytical suite that tracks student health from admission to graduation. The result is a proactive tool that suggests specific interventions (like Bridge Programs) based on data-driven triggers.

**Tools:** Power BI, Tableau, DAX, SQL, Python, Pandas.
**Domain:** Higher Education, Operations Research.
